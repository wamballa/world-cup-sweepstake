# UX Design Direction

## Feel

Premium, polished, slick, low-friction, mobile-first, social, and tournament-ready. The app should feel easier than a spreadsheet and more exciting than a static draw.

The approved high-energy visual direction is captured in `/docs/bl-091-visual-direction.md`. Use that document as the current source of truth for palette, shape language, campaign panels, header treatment, and how the BL-091 prototype direction should roll into real pages.

## Required Screens

- Admin create sweepstake.
- Sweepstake settings: name and tournament.
- Badge category display.
- Participant entry and optional emails.
- Add admins.
- Allocation review and confirmation.
- Shared sweepstake home.
- Tabs: Participants, Teams, Badges, Matches, Stats.
- Optional AI update surface.
- Loading, empty, error, stale-data, and no-email states.

## Mobile Priorities

- Admin can add many participant names quickly.
- The allocation button feels important and satisfying.
- Participants instantly see their team or teams.
- Leaderboard rank, points, badge status, and recent changes are obvious.
- Navigation stays shallow and tab-based.

## Recommended Tabs

- Participants: participant totals, allocated teams, rank movement.
- Teams: team scores, fixtures, tournament status.
- Badges: category status and current holders.
- Matches: fixtures, results, freshness labels.
- Stats: goals, clean sheets, cards, progress, scoring breakdown.

## Animation Rules

- Use motion for team allocation reveal, leaderboard movement, badge status changes, and summary reveals.
- Keep animations short, tactile, and responsive.
- Do not obscure scores, ranks, badge status, or stale-data warnings.

## Premium Signals

- Crisp typography and spacing.
- Clear hierarchy for participant, team, and badge data.
- High-quality microinteractions.
- Original visual identity.
- Thoughtful empty and error states.
- Fast mobile interactions for office and group use.
